Hexo 入门

2022-07-21

Hexo

Hexo 就是一个将markdown转译成blog的工具。基本上，只要输入一个markdown文件，Hexo就可以将其转译成一个blog

开发环境与配置

环境

Hexo不需要很复杂的开发环境，基本上只需要三个工具

  • Git
  • Node.js
  • Hexo

基本安装的话，就参考官网的一个教程即可

配置

Hexo中可修改的配置有很多，基本都放置在_config.yml里面。使用IDEA打开即可修改。

注意：Hexo使用的配置文件的格式是.yml
我们也可以自己写config，只要用这个指令把写好的config配置给hexo就可以了
$ hexo server --config custom.ymln
基本上可以配置的东西都不怎么重要，所以就看感觉配就好了没有特别需要注意的

前期准备

Github部分

首先需要使用github账号创建一个仓库，这个仓库的名称有限制条件。
必须要保证仓库的名字为 <github_username>.github.io
这一步非常关键，因为只有仓库名称正确才可以发布到online，不然就只能是本地博客

SSH部分

从本地连接到github的部分，Hexo采用的是公钥的方式。所以我们需要使用git创建一个公钥，然后把这个公钥链接给github。创建的具体方式可以参考这个文章。
得到公钥之后，就需要把公钥和GitHub连接在一起。
